# `CSH003` - Content structure

Tags: `changeset`

Aliases: `changeset-content-structure`

This rule is triggered when a changeset file contains structural quality issues: empty sections, code blocks without a language identifier, or empty list items.

## Checks

This rule validates three things:

1. **Empty sections** -- An h2 heading followed immediately by another h2 heading or the end of file with no content in between.
2. **Code blocks without language** -- A fenced code block that does not specify a language identifier (e.g., `` ``` `` instead of `` ```ts ``).
3. **Empty list items** -- A list item with no text content.

## Invalid

Empty section (no content under the heading):

```markdown
## Features

## Bug Fixes

- Fixed login issue
```

Code block without a language identifier:

````markdown
## Features

```
const x = 1;
```
````

Empty list item:

```markdown
## Features

- Added new API endpoint
-
- Fixed something
```

## Valid

```markdown
## Features

- Added new API endpoint

## Bug Fixes

- Fixed login issue
```

Code blocks with language identifiers:

````markdown
## Features

```ts
const x = 1;
```
````

## Fix

- **Empty sections**: Add content (typically a list of changes) under the heading, or remove the empty heading entirely.
- **Code blocks**: Add a language identifier after the opening fence (e.g., `` ```ts ``, `` ```json ``, `` ```bash ``).
- **Empty list items**: Add descriptive text to the list item, or remove it.

## Rationale

Empty sections and unlabeled code blocks create noise in the generated changelog and make it harder for AI agents to extract structured information. Requiring content in every section ensures that the changelog formatter produces meaningful output, and language identifiers on code blocks enable syntax highlighting and automated code analysis.
