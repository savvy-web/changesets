# `CSH002` - Required sections

Tags: `changeset`

Aliases: `changeset-required-sections`

This rule is triggered when an h2 heading in a changeset file does not match one of the 13 recognized category headings. The comparison is case-insensitive.

## Valid h2 headings

The following headings are the only valid h2-level section headings in a changeset file, listed in priority order:

| Heading | Description |
| --- | --- |
| Breaking Changes | Backward-incompatible changes |
| Features | New functionality |
| Bug Fixes | Bug corrections |
| Performance | Performance improvements |
| Documentation | Documentation changes |
| Refactoring | Code restructuring |
| Tests | Test additions or modifications |
| Build System | Build configuration changes |
| CI | Continuous integration changes |
| Dependencies | Dependency updates |
| Maintenance | General maintenance |
| Reverts | Reverted changes |
| Other | Uncategorized changes |

## Invalid

Unrecognized h2 headings trigger this rule:

```markdown
## New Stuff

- Added something

## Bugfixes

- Fixed something
```

Both `New Stuff` and `Bugfixes` are invalid. Use `Features` and `Bug Fixes` instead.

## Valid

```markdown
## Features

- Added new API endpoint

## Bug Fixes

- Fixed login issue
```

## Fix

Replace the unrecognized heading with the closest matching heading from the valid list above. Common mistakes:

- `Bugfixes` or `Bug fixes` -- use `Bug Fixes`
- `New Stuff` or `Added` -- use `Features`
- `Chores` or `Misc` -- use `Maintenance`
- `Deps` -- use `Dependencies`
- `Perf` -- use `Performance`

## Rationale

Restricting h2 headings to a fixed set of categories ensures that the changelog formatter and remark transform pipeline can reliably parse and reorder sections. It also gives AI agents a deterministic vocabulary for categorizing changes, preventing drift and inconsistency across releases.
