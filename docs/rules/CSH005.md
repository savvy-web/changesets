# `CSH005` - Dependency table format

Tags: `changeset`

Aliases: `changeset-dependency-table-format`

This rule is triggered when a `## Dependencies` section does not contain a properly structured GFM table with the required columns: Dependency, Type, Action, From, To.

## Invalid

Wrong column headers:

```markdown
## Dependencies

| Package | Version |
| :--- | :--- |
| lodash | 4.17.21 |
```

Invalid type value:

```markdown
## Dependencies

| Dependency | Type | Action | From | To |
| :--- | :--- | :--- | :--- | :--- |
| lodash | runtime | updated | 4.17.20 | 4.17.21 |
```

Valid types are: `dependency`, `devDependency`, `peerDependency`, `optionalDependency`, `workspace`, `config`.

Invalid action value:

```markdown
## Dependencies

| Dependency | Type | Action | From | To |
| :--- | :--- | :--- | :--- | :--- |
| lodash | dependency | upgraded | 4.17.20 | 4.17.21 |
```

Valid actions are: `added`, `updated`, `removed`.

Missing em dash for `added` row (From must be an em dash):

```markdown
## Dependencies

| Dependency | Type | Action | From | To |
| :--- | :--- | :--- | :--- | :--- |
| lodash | dependency | added | 4.17.20 | 4.17.21 |
```

Missing em dash for `removed` row (To must be an em dash):

```markdown
## Dependencies

| Dependency | Type | Action | From | To |
| :--- | :--- | :--- | :--- | :--- |
| lodash | dependency | removed | 4.17.20 | 4.17.21 |
```

## Valid

A correctly structured dependency table:

```markdown
## Dependencies

| Dependency | Type | Action | From | To |
| :--- | :--- | :--- | :--- | :--- |
| lodash | dependency | updated | ^4.17.20 | ^4.17.21 |
| typescript | devDependency | added | — | ~5.4.0 |
| react | peerDependency | removed | ^18.0.0 | — |
| @acme/utils | workspace | updated | 1.0.0 | 1.1.0 |
| eslint-config-acme | config | updated | 2.0.0 | 3.0.0 |
```

Version values must be semver format with an optional `~` or `^` prefix, or an em dash (U+2014: `—`) for `added` (no From) and `removed` (no To) rows.

## Fix

Ensure the `## Dependencies` section contains a GFM table with exactly five columns: Dependency, Type, Action, From, To. Use only valid type and action values. For `added` rows, set From to an em dash (`—`). For `removed` rows, set To to an em dash (`—`). Use semver versions with optional `~` or `^` prefix.

## Rationale

A structured table format for dependency changes enables tooling to parse, aggregate, sort, and collapse dependency updates automatically. The fixed column schema ensures that downstream transforms (such as `aggregate-dependency-tables`) can reliably process dependency information across multiple changesets. The em dash convention for `added` and `removed` rows makes the semantics explicit: a new dependency has no previous version, and a removed dependency has no target version.
