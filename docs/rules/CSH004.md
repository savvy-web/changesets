# `CSH004` - Uncategorized content

Tags: `changeset`

Aliases: `changeset-uncategorized-content`

This rule is triggered when content appears before the first h2 heading in a changeset file. All meaningful content must be placed under a category section (`## heading`).

HTML comments and blank lines before the first h2 are allowed and will not trigger this rule.

## Invalid

Content before the first h2 heading:

```markdown
This paragraph is not under any section.

## Features

- Added new API endpoint
```

A list before the first heading:

```markdown
- This is orphaned content

## Bug Fixes

- Fixed login issue
```

## Valid

All content is under a category heading:

```markdown
## Features

- Added new API endpoint

## Bug Fixes

- Fixed login issue
```

HTML comments before the first heading are allowed (commonly used for changeset metadata):

```markdown
<!-- This comment is fine -->

## Features

- Added new API endpoint
```

## Fix

Move the content under the appropriate category heading. If the content does not fit any existing category, place it under `## Other`.

## Rationale

The changelog formatter processes content by section. Any content that appears before the first h2 heading is effectively invisible to the pipeline and will be silently dropped from the generated changelog. This rule catches that mistake early, ensuring every piece of content is properly categorized and preserved in the final output. For AI agents, this constraint guarantees that all content is reachable via the section-based parsing model.
